# Pages Repo — CLAUDE.md
# reports.megamarketing.com.au

## Purpose
Publish client reports, dashboards, and deliverables for Mega Marketing clients.

## Folder rules
- Edit files in `public-encrypted/` ONLY — never edit `public/` directly
- `public/` is auto-generated by the encryption pre-commit hook — do not touch
- `public-encrypted/reports/` → live at reports.megamarketing.com.au/reports/
- `public-encrypted/dashboards/` → live at reports.megamarketing.com.au/dashboards/

## What to encrypt (always)
- Client performance reports (ad metrics, ROAS, CPL, revenue)
- Campaign dashboards
- Strategy docs, proposals, audit findings
- Anything with client-specific numbers

## What to leave public
- Portfolio / case studies (redacted)
- Public landing pages

## Naming convention
Files: `YYYY-MM-[client-slug]-[type].html`
Example: `2026-05-labor-care-ads-report.html`

## Deploy
- Push to main → GitHub Actions deploys in ~60 seconds
- Live URL: reports.megamarketing.com.au/[path]
- NEVER force push to main

## Commit style
Add:, Fix:, Update:, Remove: — same as workspace repo
